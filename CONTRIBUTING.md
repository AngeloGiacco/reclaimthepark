# Contributing

This is an open-source campaign. Everyone is welcome to contribute.

## How to Help

### No coding required
- **Submit an FOI:** Use the [template](comms/templates/foi-template.md) to submit requests to authorities
- **Write to your councillor:** Use the [template](comms/templates/councillor-letter.md) if you're a Westminster resident
- **Share the campaign:** Link to the website, share the blog post, retweet
- **Enter the design competition:** Watch the [briefs](competitions/briefs/) folder for open calls
- **Attend events:** Check [community/events](community/events/) for upcoming activities

### Coding & design
- **Website improvements:** See [website/README.md](website/README.md) for the tech stack and design direction
- **Data analysis:** Add notebooks to [research/analysis/](research/analysis/) — always document your data sources
- **Competition platform:** Help build the submission and voting interface in [competitions/platform/](competitions/platform/)

### Research & writing
- **Economic analysis:** Improve our models, find new data sources, challenge our assumptions
- **Blog posts:** Draft posts for the campaign blog — submit as PRs
- **Precedent research:** Add profiles of comparable sites to [research/comparables/](research/comparables/)

## Guidelines

1. **Stay on message.** We are agnostic about what replaces the car park. Don't advocate for a specific design in campaign materials.
2. **Cite everything.** Every statistic needs a source. Add sources to [research/sources/](research/sources/).
3. **Be respectful.** We don't attack people or organisations. We present a better alternative.
4. **Protect privacy.** Never commit personal data. See [comms/README.md](comms/README.md) for the privacy policy.
5. **Keep it accessible.** Website content should meet WCAG 2.1 AA. Write in plain English.

## Pull Requests

- One topic per PR
- Reference an issue if one exists
- For content changes: include sources
- For code changes: test locally before submitting
- For new data: include methodology notes

## Code of Conduct

Be kind. Be constructive. Assume good faith. If you wouldn't say it at a community meeting, don't write it in an issue.
